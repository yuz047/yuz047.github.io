Core idea

A trading agent that does less, but records more.

Greenlight is not a free-form agent that invents trades. The production track is deterministic: build a daily universe, compute features, score stocks and ETFs, select dynamic ETF exposure, allocate target weights, run risk gates, and trade only when drift, persistence, interval, turnover, and minimum-size rules agree. The AI layer reviews and explains; it cannot execute or override risk.

Architecture

Daily deterministic replay loop

The replay path mirrors daily paper mode: each trading day rebuilds data, scores, allocations, execution decisions, benchmark curves, logs, and watermarked review output.

production path
01

Market data and universe

Massive and tagged secondary data feed anchors, dynamic ETFs, current holdings, movers, relative-strength names, and liquid stock candidates.

02

Features and learning

Daily features are sliced no-lookahead. Rolling windows refresh stock and ETF score weights before each replay decision.

03

Selection and allocation

Regime, ETF discovery, candidate scores, volatility, caps, and sleeve budgets produce target weights rather than trade instructions.

04

Execution and records

Sparse execution checks drift, persistence, interval, turnover, and trade size, then writes portfolio state, benchmarks, and decision logs.

Disclosure

Research dashboard, not investment advice.

Greenlight Trader is a simulated allocation model for research and demonstration. It does not place real orders, does not manage outside capital, and should not be treated as a recommendation to buy or sell securities.
